# 2.3 Service Communication

## Synchronous Communication

Used when the caller needs an immediate response.

```text
Client
   │
API Gateway
   │
Product Service
```

Examples:

* Login
* Get Product
* Get User Profile
* Search Products
* View Order

---

## Asynchronous Communication

Used when services don't need an immediate response.

```text
Order Service
      │
OrderCreated Event
      │
RabbitMQ
      │
 ├── Inventory
 ├── Payment
 ├── Notification
 └── Analytics
```

Examples:

* Order placed
* Payment successful
* Email sent
* Inventory updated

---

# 2.4 Database Architecture

Each service owns its own database.

| Service        | Database                 |
| -------------- | ------------------------ |
| Auth           | PostgreSQL               |
| User           | PostgreSQL               |
| Product        | MongoDB                  |
| Cart           | Redis                    |
| Inventory      | PostgreSQL               |
| Order          | PostgreSQL               |
| Payment        | PostgreSQL               |
| Shipping       | PostgreSQL               |
| Review         | MongoDB                  |
| Notification   | MongoDB                  |
| Search         | Elasticsearch/OpenSearch |
| Recommendation | Neo4j                    |
| Analytics      | ClickHouse               |

**Rule:** Services never access another service's database directly. They communicate via APIs or events.
